# Техническое задание на проект "KaggleStat"

## 1. Цель проекта

Создание веб-сервиса мониторинга для участников Kaggle с целью отслеживания динамики лидербордов посредством дашбордов/графиков.

## 2. Роль пользователей

**Гость:**
-   Просмотр публичных соревнований
-   Доступ к общим графикам активности
-   Просмотр топ-лидербордов

**Зарегистрированный пользователь:**
-   Привязка Kaggle-аккаунта
-   Персонализированные дашборды
-   Настройка уведомлений о изменении позиции
-   Отслеживание конкретных участников

## 3. Модели данных

### 1. Contest (Соревнование)
Основная сущность соревнования Kaggle.
-   `competition_slug` (CharField): Слаг контеста
-   `title` (CharField): Название соревнования
-   `deadline` (DateTimeField): Срок окончания
-   `participant_count` (IntegerField): Количество участников
-   `prize` (CharField): Награда
-   `description` (CharField): Описание
-   `image_url` (CharField): Ссылка на картинку соревнования

### 2. Team (Пользователь Kaggle)
-   `team_slug` (CharField): Никнейм команды на платформе
-   `team_id` (IntegerField): Айди команды в лидерборде на платформе
-   `contests` (ManyToManyField): Связь контеста с пользователем (через модель ContestTeam)

### 3. LeaderBoard (Таблица лидеров)
-   `contest_id` (FK): Ссылка на контест
-   `saved_at` (DateTimeField): Время сохранения таблицы
-   `data` (JSONField): Данные лидерборда
-   `version` (InteferField): Версия снимка лидерборда

## 4. Ключевой функционал

**Просмотр соревнований и лидербордов.**
Пользователь может выбрать интересное ему соревнование на главной странице и посмотреть информацию о нем.

**Визуализация динамики лидерборда.**
Пользователь, отслеживающий соревнование, может видеть график изменения позиций топ-10 участников за последние 7 дней.

**Персонализированное отслеживание позиции.**
Зарегистрированный пользователь может видеть свою текущую позицию и историю её изменения, чтобы отслеживать прогресс.

**Уведомления о значимых изменениях.**
Зарегистрированный пользователь может получать уведомления, когда позиция в лидерборде сменилась в худшую сторону, чтобы своевременно реагировать на активность конкурентов.

## 5. Технический стек и интеграции

-   **Интеграция:** Использование Kaggle API
-   **Аналитика и визуализация:** pandas, chart.js
